# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Decorix, please report it privately rather than opening a public issue.

- Open a [GitHub Security Advisory](https://github.com/hermiforge/decorix/security/advisories/new) on this repository, or
- Email the maintainers with details of the issue.

Please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce (a minimal repro model/config is ideal, since Decorix is a validation library).
- The affected package(s) and version(s) (`@hermiforge-decorix/core`, `@hermiforge-decorix/cli`, or an adapter).

We will acknowledge reports as soon as possible and follow up with a fix or mitigation plan. Please give us reasonable time to address the issue before any public disclosure.

## Supported Versions

Decorix is pre-1.0. Only the latest published version of each `@hermiforge-decorix/*` package is supported with security fixes.
